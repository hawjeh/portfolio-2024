### What access policies to setup?

First we need to understand the goal - the goal is to allow NextJS to consume the blog content and images.

Hence, we know there are 2 collections to set in <b>Directus access policies</b>.

<br/>

Set <b>directus_files</b> to Public policy with <b>READ</b> access

![Screenshot1](/assets/blogs/directus-mini-blog-3/screen1.png)

<br/>
Create a new policy name <b>Presentation</b>

Set <b>blog</b> with <b>READ</b> access, <b>without</b> App and Admin Access

![Screenshot2](/assets/blogs/directus-mini-blog-3/screen2.png)

<br/>

Create a Role, name <b>Presentation</b> and <b>adopt Presentation Policy</b>

![Screenshot3](/assets/blogs/directus-mini-blog-3/screen3.png)

<br/>
Lastly, create a bot user, set

- Status = Active
- Role = Presentation
- Generate security token


![Screenshot4](/assets/blogs/directus-mini-blog-3/screen4.png)

<br/>
<b>Keep</b> the security token, that will be use for NextJS to communicate with Directus CMS.
